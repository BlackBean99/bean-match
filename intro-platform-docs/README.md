# 소개팅 운영 플랫폼 문서 패키지

이 패키지는 소개팅 운영 플랫폼의 최종 개발사항을 반영한 기획/설계 문서 모음입니다.

## 포함 문서
- `01-PRD.md` : 제품 요구사항 문서
- `02-Domain-State-FSM.md` : 도메인 모델, 상태 정의, 상태 전이
- `03-ERD-and-Schema.md` : ERD, 데이터 모델, 제약조건
- `04-API-File-Storage-and-Security.md` : API, 파일 업로드/저장, 보안
- `05-Operations-Admin-Policy.md` : 운영 정책, 관리자 화면, 배치/자동화

## 설계 핵심
- 유저 상태와 소개 건 상태를 분리
- `PROGRESSING` 유저 상태를 별도로 둬서 모집인이 즉시 이해 가능하게 설계
- 모집인(INVITOR)은 상태가 아니라 Role로 관리
- 사진은 서버 파일시스템에 저장하고 DB에는 메타데이터를 저장
- 소개 결과 확인이 끝나기 전까지 다른 소개 제안 불가
